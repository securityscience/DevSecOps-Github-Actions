Certainly! Below are examples of:

1. **An OIDC Authentication Request** (URL the app/browser would redirect the user to).
2. **An OIDC ID Token (JWT)** response returned by the Identity Provider (IDP) after successful login.

---

### ‚úÖ 1. Sample **OIDC Authentication Request** (Authorization Code Flow)

```
GET https://idp.example.com/auth
    ?response_type=code
    &client_id=my-client-id
    &redirect_uri=https%3A%2F%2Fmyapp.com%2Fcallback
    &scope=openid%20email%20profile
    &state=xyzABC123
    &nonce=abc123nonce
```

**Explanation:**

* `response_type=code`: Requesting authorization code (Authorization Code Flow)
* `client_id`: The client application's registered ID
* `redirect_uri`: Where the IDP will send the authorization code
* `scope=openid email profile`: Requests OIDC authentication and claims
* `state`: Used to prevent CSRF
* `nonce`: Used to prevent replay attacks (validated in the ID token)

---

### ‚úÖ 2. Sample **OIDC ID Token** (JWT format)

An ID token is a [JWT](https://jwt.io) containing claims about the authenticated user:

#### Header (Base64URL-decoded):

```json
{
  "alg": "RS256",
  "kid": "abc123key",
  "typ": "JWT"
}
```

#### Payload (Base64URL-decoded):

```json
{
  "iss": "https://idp.example.com",
  "sub": "248289761001",
  "aud": "my-client-id",
  "exp": 1722809037,
  "iat": 1722805437,
  "auth_time": 1722805400,
  "nonce": "abc123nonce",
  "email": "user@example.com",
  "email_verified": true,
  "name": "Jane Doe",
  "picture": "https://idp.example.com/user.jpg"
}
```

#### Signature:

Signed using the IDP's private key. Clients verify it using the public key fetched from the **JWKS endpoint** (e.g., `https://idp.example.com/.well-known/jwks.json`).

### üîÅ Token Exchange (After Auth Code Grant)

After the user authenticates, the client uses the **authorization code** to obtain tokens:

#### Token Request (HTTP POST):

```http
POST /token HTTP/1.1
Host: idp.example.com
Content-Type: application/x-www-form-urlencoded

grant_type=authorization_code
&code=SplxlOBeZQQYbYS6WxSbIA
&redirect_uri=https%3A%2F%2Fmyapp.com%2Fcallback
&client_id=my-client-id
&client_secret=my-client-secret
```

#### Token Response (JSON):

```json
{
  "access_token": "SlAV32hkKG",
  "token_type": "Bearer",
  "expires_in": 3600,
  "id_token": "eyJhbGciOiJSUzI1NiIsInR5cCI6IkpXVCJ9.ey..."
}
```

* A real JWT with a valid signature you can verify on [jwt.io](https://jwt.io)
* A sample **access token**
* Example using **PKCE** instead of client secret
